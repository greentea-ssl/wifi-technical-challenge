# Wi-Fi 非依存の時刻同期手段 — 検討メモ

> 本書は意思決定のための比較検討ドラフト。具体的部品の遅延数値・先行事例は背景調査の結果を踏まえて追って更新する（`[要裏取り]` マーク）。

## 0. 今シーズンの制約（最重要）

本書は最初「天井にIRビーコンを吊る」前提で書いたが、以下の制約が判明した：

| 制約 | 内容 |
|---|---|
| **タイムライン** | 大会まで残り**約1ヶ月** |
| **外装** | ロボット外装の変更不可（受光素子の追加配置が不可） |
| **会場設備** | 天井へのIRビーコン設置不可（会場設営時のみ可能） |
| **パドック側** | フィールド脇のパドックからの設置は可能性あり |
| **回路追加** | PCB内部への部品追加・水晶換装は可能 |
| **試作余裕** | 並行開発は可能 |

この結果、**今シーズン採用可能な「Wi-Fi非依存同期」は実質サブGHzモジュール内蔵か TCXO換装に限られる**。IR系統は来期以降の検討事項に降格する。

## 1. なぜ Wi-Fi 以外を検討するか

`docs/architecture.md` §3.1 で確認したとおり、会場相当の混雑 Wi-Fi 下では NTP/SNTP の対称RTT仮定が破綻し、ビーコン TSF も取り逃しが発生する。Wi-Fi 完全独立な同期手段を持つことには **2つの価値**がある：

1. **エンジニアリング保証の冗長化**: Wi-Fi TSF が一時的に不安定になっても ms 保証を維持する保険
2. **研究の真の ground truth**: Wi-Fi 完全独立な基準時計があれば、**TSF 計測誤差を Wi-Fi 独立の物差しで評価**できる。これが本プロジェクトの研究的価値を大きく上げる

## 2. 評価軸

| 軸 | 内容 |
|---|---|

## 3. 候補一覧

### A. IR / 可視光パルス同期（**今期: 外装制約により設置困難**）

```
[天井 IR LED ビーコン]──短パルス（例 1〜100Hz）──→ [各ロボット 受光素子＋ISR]
```

- **物理**: 光速で 12m = 40ns。事実上ゼロ伝搬遅延
- **干渉耐性**: RF とは完全直交。Wi-Fi 混雑とは無関係
- **必要HW（ロボット）**:
  - **BPV10NF**（Vishay PIN PD、940nm IRフィルタ内蔵、tr=2.5ns、~200円）
  - 反転TIA（OPA354 / LMH6629 等、GBW>100MHz）
  - 高速コンパレータ（TLV3501 等、伝搬遅延 4.5ns）
  - **TSOP38xx 等の復調ICは使用不可**（38kHzバンドパスにより数十〜数百μs遅延）
- **必要HW（基地局）**: TSAL6100 等の高出力 IR LED アレイ + TCXO駆動 PPS生成 (Si5351 + ±0.5ppm TCXO)
- **期待精度**:
  - ESP32-C5 GPIO ISR: typ 1.8〜2μs、Wi-Fi 動作時に 50〜100μs スパイク
  - **ETM→GPTimer ハードウェアキャプチャ**で SW ISR バイパス可能 → **<200ns 精度**達成可
- **先行事例**: HTC Vive Lighthouse（sync pulse 59〜139μs、Triad Semi TS3633 互換センサICあり）
- **今期実現性**: ✗ 天井設置不可、ロボット外装変更不可（上面センサ追加不可）

### A'. パドック設置IR（将来オプション）

- パドック（フィールド脇）からの IR 照射 + ロボット**側面**の透過窓 or **既存外装の IR透過性**
- 多くのプラ筐体は 940nm IR を一定割合透過する可能性あり `[机上実験で確認可能]`
- 今期は不採用、来期以降に検討

### **🆕 F. UWB (Ultra-Wideband, DW3000) — 今期の最有望候補**

```
[コートサイドの UWB アンカー基局]──ranging/sync message──→[ロボット PCB内蔵 DW3000]
```

- **物理**: UWB は 6.5/8 GHz の広帯域インパルス無線。Wi-Fi (2.4/5GHz) と**完全直交**
- **精度**:
  - **パケットタイムスタンプ精度 15.6 ps** （DW3000 HW タイマー）
  - **ISR ジッタ ±4 ns**
  - **ノード間同期 数十ns**
- **干渉耐性**: 占有帯域が広く電力密度が低いため Wi-Fi/BLE/サブGHz と相互に問題を起こさない
- **必要HW（ロボット）**:
  - **DWM3000EVB** または ESP32+DW3000 統合ボード（$30〜$50）
  - PCB に SPI で接続、アンテナは内蔵パターン or 小型チップアンテナ
  - **外装変更不要**（UWB は適度な減衰でシェルを透過）
- **必要HW（基地局側）**:
  - 同じ DW3000 を最低 1〜2 基、コートサイドに固定
  - AIPC に USB 接続（基地局時刻と AIPC RealTime をリンク）
- **コスト**: ロボット側 ~$30/台、基地局 ~$30 × 2
- **先行事例**: SSL での同期目的の使用例は**未確認 = 研究的新規性**。屋内測位用途で広く使われており、ESP32+DW3000 の作例は豊富
- **今期実現性**: **○** PCB内蔵で完結、外装変更不要、PoC 用ボードが安価
- **弱点**:
  - 基地局を 1〜2 個コートサイドに固定する必要（パドックなら可）
  - ESP-IDF の DW3000 ドライバ整備状況は要確認 `[要裏取り]`

### B. サブGHz専用無線（433 / 868 / 920 MHz）

```
[基地局: RFM69 / nRF905 / SX1262(FSK)]──短パケット──→[各ロボット 同型モジュール]
```

- **物理**: Wi-Fi 帯（2.4/5GHz）から完全に離れた帯域
- **干渉耐性**: Wi-Fi/BLE とは独立
- **SSL での既存実装**: TIGERs Mannheim は SX1280（2.4GHz）に移行、ZJUNlict は nRF24L01+。**いずれも時刻同期目的ではなく制御リンクとして**。同期目的での公開事例は確認できず
- **期待精度**:
  - SX1262 FSK: TxDone/RxDone 割込み相対ジッタ **数十μs〜100μs** → μs同期には届かない
  - LoRa: 長プリアンブルで生フレームでは ms 級。HW タイムスタンプ補正で sub-μs を実現した研究はあるが実装重い
  - **結論**: 単純実装で sub-ms 級は可能、μs級は実装次第
- **HW コスト**: 1モジュール 300〜1500円
- **ARIB STD-T108**: 1秒1パルスなら **0.1% duty に収まり日本国内運用可能**
- **位置づけ**: UWB が使えないときの第二候補

### C. 事前有線同期 + 高安定オシレータ（ドック同期）

```
[充電・調整ドック]──USB/PPS pin──→[ロボット 内部時計]
   試合中はオシレータ自走
```

- **発想**: 試合前に有線で時刻投入、試合中はオシレータの安定度に賭ける
- **10分試合での累積ドリフト試算**:

| オシレータ | 仕様 | 10分での誤差 | 具体製品例 | 単価 |
|---|---|---|---|---|
| ESP32 標準XTAL | ±10 ppm | **6 ms** ❌ | — | — |
| 汎用 TCXO | ±0.5〜2.5 ppm | **0.3〜1.5 ms** △ | NDK NT2016SF / EPSON TG-5510CA | $3〜$8 |
| 高精度 MEMS TCXO | ±0.1 ppm | **60 μs** ✓ | SiTime **SiT5356** (1-60MHz programmable) | $15〜$30 |
| OCXO | ±0.01 ppm | **6 μs** ✓ | — | 数万円 |

- **ESP32-C5 への接続**: TCXO を **XTAL_P ピンに直接入力**、内蔵発振器無効化（XTAL_N 未接続またはGND、1Vp-p AC coupling）。**C5 固有のドライブ仕様は Espressif app note 未公開** `[要裏取り]`
- **弱点**: 温度変化に弱い、計測ラン長に上限、試合中の再同期が不便

### D. ハイブリッド（D = A or B + 弱いオシレータ）★最強の現実解

- 試合中 1Hz 程度の**補正パルス**を A または B で配信
- ロボット側は補正パルス間を**標準XTAL**で自走 → 1秒で 10〜20μs ドリフトに収まる
- 高精度オシレータ不要、補正パルスの安定度で全体精度が決まる
- **A (IR) + ESP32標準XTAL** が**コスト・性能・実装性のスイートスポット**になる見込み

### E. 不採用候補

| 手段 | 不採用理由 |
|---|---|
| 超音波 | 音速 343 m/s で 12m = **35.3ms** の伝搬遅延。位置補正は本末転倒 |
| Bluetooth/BLE | 2.4GHz、Wi-Fi と同じ混雑問題 |
| 磁界結合 | 距離が短い・ロボット間混信 |
| GNSS リピータ | 電波法上グレー、会場運営許可必要 |
| 電灯線通信 | 電池駆動のロボットなので不可 |
| SSL-Vision フレーム同期 | ロボット側にカメラが必要になり過剰 |
| **JJY (LF 時報電波 40/60 kHz)** | (1) 受信PPS ジッタ **~10 ms** で要求精度に3桁不足、(2) 鉄骨大型会場で屋内受信が壊滅的、(3) **NICT の stratum-1 NTP (ntp.nict.jp)** が同じ原子時計を上流とするため、LAN 内 NTP を使えば JJY より上流の時刻が既に取れている |
| **ESP-NOW（同期信号として）** | プロトコルは軽量だが **PHY は通常 Wi-Fi と同じ 2.4/5GHz**。受信コールバックは Wi-Fi タスクから呼ばれ 1〜数ms のスケジューリングジッタ。promiscuous + rx_ctrl で μs を狙っても結局 Wi-Fi PHY の受信イベントなので TSF と同じ混雑問題を受ける。**ただし計測ログ返送路としては有用**（精度不要・別チャネル運用可、§ architecture.md 4.3 で採用済み） |
| ~~UWB (DW3000等)~~ | **再評価で最有望候補に格上げ → §3.F に独立記載** |

## 5. 推奨方針（制約反映版）

### 今シーズン（残り1ヶ月）— **Wi-Fi TSF 本線 + 軽量な保険**
- `docs/architecture.md` の Wi-Fi TSF チェーンを取り切ることに**最優先で集中**
- 並行で**低コスト・低リスク**の保険のみ追加：
  - **C. TCXO 換装の評価**: ESP32-C5 の XTAL_P に TCXO 入力可能か実機で確認。SiT5356 (±0.1ppm, $15-30) または NDK NT2016SF (±0.5ppm, $3-8) を 1〜2台のロボットで試作。**1個換装あたり PCB 改造作業 1時間程度**
- 大きな新規サブシステム（UWB, IR, サブGHz）は**今シーズンは入れない**

### 大会直後 〜 来期初期 — **🆕 UWB DW3000 を研究用 ground truth として導入**
- ESP32+DW3000 統合ボードまたは DWM3000EVB をロボット PCB に増設
- コートサイドに UWB アンカー基局を 1〜2 基設置
- TSF と UWB の両方で同一パケットに打刻 → **TSF 誤差を UWB という ns 精度の真値で評価**
- これが本プロジェクトの**研究的差別化点**（SSL で時刻同期を発表した先行事例なし）
- ESP-IDF 用 DW3000 ドライバの整備状況を Phase 2 初期に確認

### 来期 〜 中期 — **サブGHz補正パルスをオプション併用 or UWB に統一**
- UWB の動作実績が良ければそちらに統一
- UWB アンカー設置が大会会場で困難な場合のみ、サブGHz (B) を補助で残す

### 長期（次世代ロボット改修時）— **パドック IR をオプション化**
- 外装次回設計時に IR 透過窓 or 受光素子配置位置を確保
- 既製外装の **940nm IR 透過性検証**（机上実験：シェルに IR LED 照射しフォトダイオードで受光）は今期中の半日タスクで実施可能

### 今すぐ着手すべき小タスク
